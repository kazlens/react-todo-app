# Отчет по мини-проекту React To-Do App

## Идея проекта

Идея проекта - сделать приложение для списка задач. Пользователь может добавлять задачи, отмечать их выполненными, редактировать, удалять, искать по названию и фильтровать по статусу.

Еще в проект добавлена отдельная страница, где данные загружаются с внешнего API. Это нужно, чтобы показать работу React с запросами к серверу.

## Структура компонентов

- `App.jsx` - главный компонент. В нем находится список задач, тема, поиск, фильтр и маршруты.
- `TasksPage.jsx` - страница со списком задач.
- `ApiPage.jsx` - страница с задачами из Placeholder API.
- `Form.jsx` - форма добавления задачи.
- `TaskControls.jsx` - поиск и фильтрация задач.
- `List.jsx` - таблица задач.
- `ListItem.jsx` - одна задача в таблице.

Компоненты разделены так, чтобы каждый отвечал за свою часть интерфейса.

## Где используется useState

`useState` используется для данных, которые меняются во время работы приложения.

В `App.jsx`:

- `tasks` - список задач;
- `theme` - выбранная тема;
- `searchQuery` - строка поиска;
- `statusFilter` - фильтр по статусу.

В `Form.jsx`:

- `taskName` - название новой задачи;
- `error` - текст ошибки.

В `ListItem.jsx`:

- `taskName` - название задачи во время редактирования.

В `ApiPage.jsx`:

- `todos` - задачи, полученные с API;
- `loading` - загрузка данных;
- `error` - ошибка загрузки.

## Где используется useEffect

В `App.jsx` `useEffect` сохраняет выбранную тему:

```jsx
useEffect(() => {
  localStorage.setItem(THEME_STORAGE_KEY, theme)
}, [theme])
```

Еще один `useEffect` сохраняет задачи в `localStorage`:

```jsx
useEffect(() => {
  localStorage.setItem(TASKS_STORAGE_KEY, JSON.stringify(tasks))
}, [tasks])
```

В `ApiPage.jsx` `useEffect` загружает данные с API:

```jsx
useEffect(() => {
  fetch('https://jsonplaceholder.typicode.com/todos?_limit=10')
    .then((response) => response.json())
    .then((data) => {
      setTodos(data)
    })
}, [])
```

Пустой массив зависимостей значит, что запрос выполняется один раз при открытии страницы.

## Где используются props

`props` используются для передачи данных и функций между компонентами.

Основное состояние задач находится в `App.jsx`. Дальше оно передается в `TasksPage.jsx`, а потом в `Form.jsx`, `TaskControls.jsx`, `List.jsx` и `ListItem.jsx`.

Через props передаются:

- список задач;
- функция добавления задачи;
- функция удаления задачи;
- функция изменения статуса;
- функция сохранения названия;
- поиск и фильтр.

Так реализовано общее состояние: данные хранятся в главном компоненте, а дочерние компоненты получают их через props.

## Работа с API

Работа с API сделана в `ApiPage.jsx`.

Используется запрос:

```jsx
fetch('https://jsonplaceholder.typicode.com/todos?_limit=10')
```

Приложение получает 10 тестовых задач с сервиса JSONPlaceholder. Потом данные записываются в состояние `todos` и выводятся на странице.

На странице есть:

- сообщение загрузки;
- сообщение ошибки;
- список полученных задач.

Для каждой задачи показывается `id`, название и статус: `Готово` или `В работе`.

## Routing

Routing сделан с помощью `react-router-dom`.

В `main.jsx` приложение обернуто в `BrowserRouter`.

В `App.jsx` используются:

- `Routes`;
- `Route`;
- `NavLink`.

В проекте есть два маршрута:

- `/` - страница задач;
- `/api` - страница API.

Переход между страницами выполняется через кнопки `Задачи` и `API`.

## Тема приложения

В приложении есть светлая и темная тема. Тема хранится в состоянии `theme` и сохраняется в `localStorage`.

Для оформления используется Tailwind CSS. Темная тема сделана через класс `dark` и префикс `dark:`.

Примеры классов:

```jsx
dark:bg-gray-900
dark:text-white
dark:border-white/20
```

## Основная логика приложения

При запуске приложение проверяет `localStorage`. Если там есть сохраненные задачи, они загружаются. Если данных нет, используется начальная задача.

Пользователь может:

- добавить задачу;
- отметить задачу выполненной;
- изменить задачу;
- удалить задачу;
- искать задачи;
- фильтровать задачи;
- переключать тему;
- открыть страницу API.

При добавлении задачи создается новый объект:

```jsx
const task = {
  id: Date.now(),
  name,
  editing: false,
  done: false
}
```

Фильтрация работает по названию задачи и выбранному статусу. После изменения задач список сохраняется в `localStorage`, поэтому задачи остаются после перезагрузки страницы.
